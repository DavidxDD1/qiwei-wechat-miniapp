# 手工测试用例

## TC-PRICE-001
- 场景：男陪顶尖机密 1 小时
- 步骤：服务页 -> 男陪价格表 -> 顶尖单价 -> 机密行 -> 1小时列
- 预期：底部价格显示 `100`

## TC-PRICE-002
- 场景：女陪娱乐绝密包月
- 步骤：服务页 -> 女陪价格表 -> 娱乐单价 -> 绝密 -> 包月10小时/天
- 预期：底部价格显示 `19199`

## TC-PRICE-003
- 场景：护航体验单
- 步骤：服务页 -> 护航单 -> 体验单 -> 128保888
- 预期：底部价格显示 `128`

## TC-CART-001
- 场景：加购物车
- 步骤：选中任一价目 -> 加入购物车
- 预期：购物车增加一项

## TC-ORDER-001
- 场景：无手机号提交
- 步骤：下单页不填手机号 -> 提交
- 预期：提示手机号错误

## TC-ORDER-002
- 场景：正常创建订单
- 步骤：购物车有项 -> 填完整信息 -> 提交
- 预期：跳转支付/说明页，显示订单号

## TC-PAY-001
- 场景：支付禁用
- 步骤：进入支付/说明页
- 预期：显示人工确认说明，不调用支付

## TC-PROFILE-001
- 场景：个人中心占位
- 步骤：进入个人中心
- 预期：游客态头像、会员占位、消息占位、评价占位可见
