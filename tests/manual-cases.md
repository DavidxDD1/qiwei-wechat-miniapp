# 手工测试用例

## TC-PRICE-001

- 场景：男陪顶尖机密 1 小时
- 步骤：服务页 -> 男陪价格表 -> 顶尖单价 -> 机密 -> 1小时
- 预期：底部价格栏显示 `100`

## TC-PRICE-002

- 场景：女陪娱乐绝密包月
- 步骤：服务页 -> 女陪价格表 -> 娱乐单价 -> 绝密 -> 包月10小时/天
- 预期：底部价格栏显示 `19199`

## TC-PRICE-003

- 场景：护航体验单
- 步骤：服务页 -> 护航单 -> 体验单 -> `128保888`
- 预期：底部价格栏显示 `128`

## TC-CART-001

- 场景：加入购物车
- 步骤：选中任意价格单元格 -> 点击“加入购物车”
- 预期：购物车新增一项

## TC-CART-002

- 场景：购物车勾选和删除
- 步骤：进入购物车 -> 取消勾选一项 -> 再删除该项
- 预期：总价实时变化，删除后列表更新

## TC-ORDER-001

- 场景：无手机号提交
- 步骤：购物车或直购进入下单页 -> 不填手机号 -> 提交
- 预期：Toast 提示手机号错误

## TC-ORDER-002

- 场景：正常创建订单
- 步骤：选择服务项 -> 填完整联系人信息 -> 勾选服务说明 -> 提交
- 预期：跳转支付说明页，并显示订单号和应付金额

## TC-PAY-001

- 场景：支付禁用
- 步骤：进入支付说明页
- 预期：显示“在线支付暂未开通，请联系客服确认订单”类说明，不调用真实支付

## TC-ORDERS-001

- 场景：订单列表筛选
- 步骤：创建订单后进入订单页 -> 切换“全部 / 待确认 / 已确认 / 已取消”
- 预期：列表按状态筛选展示

## TC-PROFILE-001

- 场景：个人中心占位
- 步骤：进入个人中心
- 预期：游客态资料、会员占位、消息占位、评价占位均可见
